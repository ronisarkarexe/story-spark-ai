# useCollaboration Hook

## Overview

The `useCollaboration` hook provides real-time collaborative editing for story writing sessions using socket.io. Multiple users can join the same room and edit the story simultaneously, with live updates streamed to all participants.

## Location

`frontend/src/hooks/useCollaboration.ts`

## Usage

```tsx
import { useCollaboration } from "../hooks/useCollaboration";

function CollaborativeEditor({ roomId }: { roomId: string }) {
  const {
    room,
    loading,
    error,
    typingUsers,
    isAiThinking,
    addText,
    emitTyping,
    stopTyping,
    requestAiContinue,
  } = useCollaboration({ roomId });

  if (loading) return <div>Loading room...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <div>
      <p>{room?.participants.length} participants</p>
      {Object.values(typingUsers).map((name) => (
        <span key={name}>{name} is typing...</span>
      ))}
      {isAiThinking && <span>AI is thinking...</span>}
      <button onClick={() => addText("Hello world")}>Add Text</button>
      <button onClick={emitTyping}>Typing</button>
      <button onClick={stopTyping}>Stop Typing</button>
      <button onClick={requestAiContinue}>AI Continue</button>
    </div>
  );
}
```

## Options

```ts
interface UseCollaborationOptions {
  /** The ID of the collaboration room to join. Undefined means no connection. */
  roomId: string | undefined;
  /** Optional callback invoked when a collaboration error occurs. */
  onError?: (message: string) => void;
}
```

## Return Value

```ts
interface UseCollaborationReturn {
  /** The current room state, including participants, story chunks, and metadata. null when not connected. */
  room: CollabRoom | null;
  /** True while the hook is connecting and fetching initial room state. */
  loading: boolean;
  /** Error message string, or null when there is no error. */
  error: string | null;
  /** Map of userId to username for users currently typing. */
  typingUsers: Record<string, string>;
  /** True while the AI is generating content for the room. */
  isAiThinking: boolean;
  /** Append a text chunk to the story in the room. */
  addText: (text: string) => void;
  /** Emit a typing indicator for the current user. Auto-clears after 300ms. */
  emitTyping: () => void;
  /** Stop the current user's typing indicator immediately. */
  stopTyping: () => void;
  /** Request the AI to continue generating the story from the current state. */
  requestAiContinue: () => void;
}
```

## Exported Types

### `Participant`

Represents a user currently in the collaboration room.

```ts
interface Participant {
  userId: string;
  username: string;
  color: string;   // Assigned cursor/selection color for visual differentiation
  socketId: string;
}
```

### `StoryChunk`

A single segment of story text added by a participant or the AI.

```ts
interface StoryChunk {
  authorId: string;
  authorName: string;
  color: string;
  text: string;
  isAI: boolean;    // true when this chunk was generated by the AI
  timestamp: Date;
}
```

### `CollabRoom`

The full state of a collaboration room.

```ts
interface CollabRoom {
  roomId: string;
  createdBy: string;       // userId of the user who created the room
  participants: Participant[];
  story: StoryChunk[];     // ordered list of story segments
  createdAt: Date;
}
```

## Socket Events

The hook connects to the socket namespace `/collab` with the following event flow:

**Client -> Server:**
- `collab:join_room` — join a room by roomId
- `collab:get_room` — fetch the current room state (expects callback)
- `collab:add_text` — append a text chunk to the story
- `collab:typing` — signal that the current user is typing
- `collab:stop_typing` — signal that the current user stopped typing
- `collab:ai_continue` — request AI to continue the story

**Server -> Client:**
- `collab:room_updated` — room metadata changed (e.g. user joined/left)
- `collab:story_updated` — story chunks were updated
- `collab:user_typing` — another user started typing
- `collab:user_stop_typing` — another user stopped typing
- `collab:ai_thinking` — AI has started generating
- `collab:error` — an error occurred in the collaboration session

## Connection Lifecycle

1. When `roomId` is provided, the hook connects to the socket server and joins the room.
2. It immediately fetches the current room state via `collab:get_room`.
3. While connected, it listens for real-time updates (`story_updated`, `user_typing`, etc.).
4. When `roomId` changes to `undefined` or the component unmounts, the socket is disconnected and cleaned up.
5. A typing indicator automatically clears 300ms after `emitTyping` is called if `stopTyping` is not called first.

## Dependencies

- `socket.io-client` — real-time communication
- `@/services/auth.service` — retrieves the JWT token for socket authentication
- `@/helpers/socket-url` — resolves the socket server URL from the current environment
